Specifications

DJENG stands for Dynamic Java Encoder (NextGen). It is the next generation of byte stream codec framework following another project, Jen8583. Although the original intention was to provide a framework for creating codecs to encode/decode ISO8583/AS2805 messages, it evolved into a dynamic framework which can be used to easily create a codec for any message format.

The key components of djeng are the xml configuration, the message codec and the message. Message codecs are built based on the xml configuration defined by the user. These message codec then performs the heavy lifting of encoding/decoding between message and from byte arrays. Lastly, the message provides a way for users to query message elements after decoding a byte array and to build messages prior to encoding to byte arrays.
