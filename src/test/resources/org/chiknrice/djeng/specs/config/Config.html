XML Configuration

Djeng uses an xml configuration to define the message structure and codecs (encoder/decoder) to be used for each message element. The config file is then validated against a comprehensive schema (djeng.xsd) which enforces most of the rules that defines a valid message codec configuration.

There are just a few elements that comprises the configurations:

  • codecs
  • filter-codec
  • element-codec
  • composite-codec
  • message-elements
  • element
  • composite

Djeng requires at least a minimum configuration which respects the following rules before any codecs are created:

  • Unique Codec ID
  • Valid Codec Reference
  • Unique Element Index
  • Valid Filter Reference
  • Valid Identifiers
