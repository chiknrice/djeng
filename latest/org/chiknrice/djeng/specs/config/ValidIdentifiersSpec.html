    Valid Identifiers

    Project maintained by chiknriceTheme by mattgraham

    Values of the following are based on non-empty-no-space-token type which requires a value without any spaces even in the middle of characters.

    • id
    • codec
    • class
    • filter
    On the other hand, the index attribute is more strict. Valid values are limited to alphanumeric characters and the hyphen (-).

    The following configuration would result in cvc-pattern-valid: Value 'my string' is not facet-valid with respect to pattern '[^\s]+' for type 'non-empty-no-space-token'. 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <element-codec id="my string" class="org.chiknrice.djeng.fin.StringCodec"/>
        <composite-codec id="message" class="org.chiknrice.djeng.fin.PlainCompositeCodec"/>
    </codecs>
    <message-elements codec="message">
        <element index="1" codec="my string"/>
    </message-elements>

</config>

    The following configuration would result in cvc-pattern-valid: Value 'a$1' is not facet-valid with respect to pattern '[a-z,A-F,0-9,-]+' for type 'index-type'. 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <element-codec id="string" class="org.chiknrice.djeng.fin.StringCodec"/>
        <composite-codec id="message" class="org.chiknrice.djeng.fin.PlainCompositeCodec"/>
    </codecs>
    <message-elements codec="message">
        <element index="a$1" codec="string"/>
    </message-elements>

</config>
