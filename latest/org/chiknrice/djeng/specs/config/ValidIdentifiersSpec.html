    Valid Identifiers

    Project maintained by chiknriceTheme by mattgraham

    Values of id and codec attributes are based on non-empty-no-space-token type which requires a value without any spaces. On the other hand, the index attribute is a bit more strict. Valid values does not permit spaces or the dot (.) character.

    The following configuration would result in cvc-pattern-valid: Value 'my string' is not facet-valid with respect to pattern '[^\s]+' for type 'non-empty-no-space-token'. 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <element-codec id="my string" class="org.chiknrice.djeng.fin.StringCodec"/>
        <composite-codec id="message" class="org.chiknrice.djeng.CompositeCodec"/>
    </codecs>
    <message-elements codec="message">
        <element index="1" codec="my string"/>
    </message-elements>

</config>

    The following configuration would result in cvc-pattern-valid: Value 'a.1' is not facet-valid with respect to pattern '[^\s.]+' for type 'index-type'. 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <element-codec id="string" class="org.chiknrice.djeng.fin.StringCodec"/>
        <composite-codec id="message" class="org.chiknrice.djeng.CompositeCodec"/>
    </codecs>
    <message-elements codec="message">
        <element index="a.1" codec="string"/>
    </message-elements>

</config>
