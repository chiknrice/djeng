    Valid Codec Reference

    Every element (including message-elements) should have its codec attribute reference an existing codec.

    Project maintained by chiknriceTheme by mattgraham

    If the reference doesn't exist like below: 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <element-codec id="y" class="org.chiknrice.djeng.fin.StringCodec"/>
        <composite-codec id="x" class="org.chiknrice.djeng.fin.PlainCompositeCodec"/>
    </codecs>
    <message-elements codec="x">
        <element index="1" codec="a"/>
    </message-elements>

</config>

    Then building the config would result in Key 'element-codec-id-ref' with value 'a' not found for identity constraint of element 'config'.
