    Element Reference

    Project maintained by chiknriceTheme by mattgraham

    All element should have its codec attribute reference an existing element-codec.

    codec referencing a composite-codec

    The following configuration would result in Key 'element-codec-id-ref' with value 'x' not found for identity constraint of element 'config'. 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <element-codec id="y" class="org.chiknrice.djeng.fin.StringCodec"/>
        <composite-codec id="x" class="org.chiknrice.djeng.fin.PlainCompositeCodec"/>
    </codecs>
    <message-elements codec="x">
        <element index="1" codec="x"/>
    </message-elements>

</config>

    codec referencing a non existing element-codec

    The following configuration would result in Key 'element-codec-id-ref' with value 'a' not found for identity constraint of element 'config'. 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <element-codec id="y" class="org.chiknrice.djeng.fin.StringCodec"/>
        <composite-codec id="x" class="org.chiknrice.djeng.fin.PlainCompositeCodec"/>
    </codecs>
    <message-elements codec="x">
        <element index="1" codec="a"/>
    </message-elements>

</config>
