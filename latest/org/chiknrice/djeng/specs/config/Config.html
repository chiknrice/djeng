    XML

    As djeng models message elements into a hierarchical structure, it makes sense to use XML to define the configuration. The xml configuration defines codecs and the structure of message elements corresponding to the arrangement specific to a message format. The configuration file is then validated against a comprehensive schema (djeng.xsd) which enforces most of the rules that defines a valid configuration.

    XML Elements

    Complex message formats can easily be defined using the small set of xml elements.

    • codecs
    • filter-codec
    • element-codec
    • composite-codec
    • message-elements
    • element
    • composite

    Schema Validation

    Djeng requires a minimum configuration which respects the following rules before any codecs are created:

    • Each codec (including filters) should have a unique id
    • Elements should reference an existing codec
    • Each element within a composite should have a unique index
    • A codec should reference a valid filter
    • Identifiers and attribute values should meet the restrictions
