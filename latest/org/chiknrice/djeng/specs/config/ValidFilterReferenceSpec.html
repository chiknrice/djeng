    Valid Filter Reference

    Every codec (or even filter) using a filter should have their filter attribute reference an existing filter.

    Project maintained by chiknriceTheme by mattgraham

    If the reference doesn't exist like below: 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <filter-codec id="lvar" class="org.chiknrice.djeng.fin.LengthPrefixCodec"/>
        <element-codec id="string" class="org.chiknrice.djeng.fin.StringCodec" filter="f"/>
        <composite-codec id="message" class="org.chiknrice.djeng.fin.PlainCompositeCodec"/>
    </codecs>
    <message-elements codec="message">
        <element index="1" codec="string"/>
    </message-elements>

</config>

    Then building the config would result in Key 'filter-codec-id-ref' with value 'f' not found for identity constraint of element 'config'.
