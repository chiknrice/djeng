    Unique Element Index

    An sub-element's index attribute should be unique within the message-elements as well as any composite. Both element and composite are considered sub-element.

    Project maintained by chiknriceTheme by mattgraham

    Below configuration is an example where two elements under message-elements are having the same index attribute: 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <element-codec id="string" class="org.chiknrice.djeng.fin.StringCodec"/>
        <composite-codec id="message" class="org.chiknrice.djeng.fin.PlainCompositeCodec"/>
    </codecs>
    <message-elements codec="message">
        <element index="1" codec="string"/>
        <element index="1" codec="string"/>
    </message-elements>

</config>

    Which results in Duplicate unique value [1] declared for identity constraint of element "message-elements".

    Below configuration is an example where an element and a composite under message-elements are having the same index attribute: 

    <?xml version="1.0" encoding="UTF-8"?>
<config xmlns="http://www.chiknrice.org/djeng">

    <codecs>
        <element-codec id="string" class="org.chiknrice.djeng.fin.StringCodec"/>
        <composite-codec id="c" class="org.chiknrice.djeng.fin.PlainCompositeCodec"/>
        <composite-codec id="message" class="org.chiknrice.djeng.fin.PlainCompositeCodec"/>
    </codecs>
    <message-elements codec="message">
        <element index="1" codec="string"/>
        <element index="4" codec="string"/>
        <composite index="4" codec="c">
            <element index="1" codec="string"/>
        </composite>
    </message-elements>

</config>

    Which in turn results in Duplicate unique value [4] declared for identity constraint of element "message-elements".
